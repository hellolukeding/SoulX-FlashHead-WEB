# 系统架构文档

## 1. 系统概述

实时数字人平台基于 SoulX-FlashHead 模型，提供实时音频驱动的数字人视频生成服务。

## 2. 架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    前端层                          │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐  │
│  │  桌面应用  │  │  Web 应用  │  │  移动端    │  │
│  └────────────┘  └────────────┘  └────────────┘  │
└─────────────────────────────────────────────────────┘
                    ↕ WebSocket/HTTP
┌─────────────────────────────────────────────────────┐
│                   API 网关层                        │
│  ┌────────────┐  ┌────────────┐                   │
│  │  REST API  │  │ WebSocket  │                   │
│  └────────────┘  └────────────┘                   │
└─────────────────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────────────────┐
│                  业务逻辑层                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │ 会话管理  │  │ 任务调度  │  │ 状态管理  │        │
│  └──────────┘  └──────────┘  └──────────┘        │
└─────────────────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────────────────┐
│                 推理引擎层                          │
│  ┌──────────────────────────────────────────────┐  │
│  │  SoulX-FlashHead 推理引擎                    │  │
│  │  - 音频处理 (wav2vec2)                       │  │

│  │  - 流式输出                                   │  │
│  └──────────────────────────────────────────────┘  │

│  └──────────┘  └──────────┘  └──────────┘        │
└─────────────────────────────────────────────────────┘
```

### 2.2 数据流

```
用户 → 音频采集 → WebSocket → 音频缓冲 → FlashHead推理
                                              ↓
用户 ← 视频播放 ← WebSocket ← 视频队列 ← VAE解码
```

## 3. 核心模块

### 3.1 推理引擎模块

**职责**：
- 模型加载和管理
- 音频特征提取
- 视频生成
- 性能监控

**接口**：
```python
class InferenceEngine:
    def load_model(model_type: str)
    def process_audio(audio_data: bytes) -> AudioEmbedding
    def generate_video(audio_embedding: AudioEmbedding) -> VideoFrames
    def get_performance_metrics() -> Metrics
```

### 3.2 流媒体处理模块

**职责**：
- 音频流缓冲管理
- 视频帧队列
- 实时传输
- 延迟优化

**接口**：
```python
class StreamProcessor:
    def add_audio_chunk(chunk: AudioChunk)
    def get_video_frame() -> VideoFrame
    def start_stream()
    def stop_stream()
```

### 3.3 会话管理模块

**职责**：
- 多会话并发
- 资源分配
- 状态维护
- 清理策略

**接口**：
```python
class SessionManager:
    def create_session(config: SessionConfig) -> SessionID
    def close_session(session_id: SessionID)
    def get_session_status(session_id: SessionID) -> SessionStatus
    def list_active_sessions() -> List[SessionInfo]
```

## 4. 技术选型

### 4.1 后端技术栈

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| Web 框架 | FastAPI | 高性能、异步支持、自动文档 |
| 推理框架 | PyTorch | SoulX-FlashHead 基于 PyTorch |
| 音频处理 | librosa + wav2vec2 | 成熟、高效 |
| 通信协议 | WebSocket + HTTP | 实时性 + 兼容性 |
| 缓存 | Redis | 高性能、支持队列 |
| 异步处理 | asyncio | Python 原生、高效 |

### 4.2 前端技术栈

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 框架 | React 19 | 生态成熟、性能优秀 |
| 语言 | TypeScript | 类型安全、开发效率高 |
| UI 库 | Ant Design | 组件丰富、设计规范 |
| 样式 | TailwindCSS | 快速开发、按需加载 |
| 桌面框架 | Tauri | 轻量、安全、性能好 |
| 状态管理 | React Hooks | 简单、高效 |
