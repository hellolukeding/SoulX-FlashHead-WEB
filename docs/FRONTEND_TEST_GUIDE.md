# 前端测试页面使用指南

**文档版本:** 1.0
**更新日期:** 2026-03-05

---

## 📋 概述

本项目提供了一个完整的前端测试页面，用于测试数字人对话系统的完整功能。

### 功能特性

- ✅ WebSocket 实时通信
- ✅ 文本对话测试
- ✅ 语音录制和识别
- ✅ 实时视频显示
- ✅ AI 语音播放
- ✅ 对话历史记录
- ✅ 系统日志显示

---

## 🚀 快速开始

### 方法一：使用启动脚本（推荐）

#### Linux / macOS
```bash
cd /opt/digital-human-platform
./start.sh
```

#### Windows
```cmd
cd C:\path\to\digital-human-platform
start.bat
```

启动脚本会：
1. 自动激活虚拟环境
2. 检查并安装依赖
3. 启动后端服务
4. 显示测试页面地址

---

### 方法二：手动启动

#### 1. 启动后端服务
```bash
cd /opt/digital-human-platform/backend
source venv/bin/activate
python -m app.main
```

#### 2. 打开测试页面
在浏览器中打开：
```
file:///opt/digital-human-platform/frontend/test_client.html
```

或使用本地服务器（推荐）：
```bash
cd /opt/digital-human-platform/frontend
python -m http.server 8080
```

然后访问：http://localhost:8080/test_client.html

## 📖 使用说明

### 1. 连接服务器

1. 在"服务器地址"输入框中确认 WebSocket URL：
   ```
   ws://localhost:8000/ws?token=test
   ```

2. 点击 **"🔌 连接服务器"** 按钮

3. 等待状态指示器变为绿色（已连接）

---

### 2. 创建会话

1. 确认"参考图像"输入框中有 Base64 编码的图像数据
   - 默认已提供一个 1x1 像素的测试图像
   - 你可以替换为自己的图像 Base64 编码

2. 点击 **"🎬 创建会话"** 按钮

3. 等待"会话创建成功"提示

---

### 3. 发送文本消息

1. 在"输入文本"输入框中输入你想说的话
   - 默认文本："你好，请介绍一下自己"

2. 点击 **"💬 发送文本"** 按钮

3. 观察对话历史中的 AI 回复

---

### 4. 录音对话

1. 点击 **"🎤 录音"** 按钮

2. 浏览器会请求麦克风权限，点击"允许"

3. 对着麦克风说话

4. 再次点击按钮（显示"⏹️ 停止录音"）

5. 系统会自动：
   - 发送音频到服务器
   - 进行语音识别（ASR）
   - 生成 AI 回复
   - 合成语音（TTS）
   - 生成数字人视频

## 🎨 界面说明

### 状态栏
- **状态指示器**: 显示连接状态
  - 🔴 红色：未连接
  - 🟢 绿色：已连接
- **会话 ID**: 显示当前会话的 ID

### 视频显示区
- 显示实时生成的数字人视频
- 等待视频流时显示占位符

### 对话历史
- 👤 用户：蓝色背景，左侧边框
- 🤖 AI 助手：紫色背景，左侧边框
- 支持流式显示 AI 回复

### 控制面板
- **服务器地址**: WebSocket 服务器 URL
- **参考图像**: Base64 编码的参考图像
- **输入文本**: 要发送给 AI 的文本

### 操作按钮
- **🔌 连接服务器**: 建立 WebSocket 连接
- **🔌 断开连接**: 关闭 WebSocket 连接
- **🎬 创建会话**: 创建新的对话会话
- **💬 发送文本**: 发送文本消息
- **🎤 录音**: 开始/停止录音
- **🗑️ 清空日志**: 清空系统日志

### 系统日志
- 显示所有操作和事件
- 不同颜色表示不同级别：
  - 🔵 蓝色：信息
  - 🟢 绿色：成功
  - 🔴 红色：错误
  - 🟡 黄色：警告

## ⚠️ 注意事项

### 1. 浏览器兼容性

**推荐浏览器:**
- ✅ Chrome 90+
- ✅ Edge 90+
- ✅ Firefox 88+
- ✅ Safari 14+

**必需功能:**
- WebSocket 支持
- MediaRecorder API（录音功能）
- Web Audio API（音频播放）
- Canvas API（视频显示）

---

### 2. 参考图像

**默认测试图像:**
- 1x1 像素的红色 PNG
- Base64 编码已预填充
- 仅用于测试，实际使用需要替换为真实图像

**如何获取 Base64:**
```bash
# 将图像转换为 Base64
base64 -i image.png -o image.txt

# 或使用在线工具
# https://www.base64-image.de/
```

---

### 3. 麦克风权限

**首次使用录音功能时:**
1. 浏览器会请求麦克风权限
2. 点击"允许"
3. 如果拒绝，需要在浏览器设置中手动允许

**常见问题:**
- 麦克风被其他应用占用
- 浏览器权限被拒绝
- 系统麦克风设置问题

---

### 4. 音频播放

**音频格式:**
- 格式：WAV
- 采样率：16kHz
- 编码：PCM

**如果音频无法播放:**
1. 检查浏览器控制台错误
2. 确认音频格式正确
3. 尝试刷新页面

## 🐛 故障排除

### 问题 1: 无法连接服务器

**错误信息:** `WebSocket connection failed`

**解决方案:**
1. 确认后端服务已启动
2. 检查服务器地址是否正确
3. 检查防火墙设置
4. 查看浏览器控制台错误

---

### 问题 2: 会话创建失败

**错误信息:** `会话创建失败`

**解决方案:**
1. 检查参考图像是否有效
2. 确认 Base64 编码正确
3. 查看系统日志中的错误信息

---

### 问题 3: 录音无法使用

**错误信息:** `无法访问麦克风`

**解决方案:**
1. 检查浏览器权限
2. 确认麦克风未被占用
3. 尝试使用 HTTPS（某些浏览器要求）

---

### 问题 4: 视频不显示

**可能原因:**
1. 视频流未正确发送
2. Canvas 未正确渲染
3. 视频格式问题

**解决方案:**
1. 检查浏览器控制台
2. 查看系统日志
3. 确认后端视频生成正常

## 🎯 高级用法

### 1. 自定义服务器地址

```

ws://localhost:8000/ws?token=test

```
